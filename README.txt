В этом репо обе практические работы

Практическая работа 1: Создание веб-приложения
Вариант 2: Пользователи и организации

В проекте реализована система управления связями между пользователями и организациями. Основные сущности:
— Организация (название, описание)
— Должность (название)
— Членство — связь пользователя с организацией на определённой должности

Особенности:
— Один пользователь может состоять в нескольких организациях, но только на одной должности в каждой.
— При удалении должности у пользователя в интерфейсе отображается «Сотрудник».
— Реализован полный CRUD через CreateView, UpdateView, DeleteView и другие классы Django.
— Есть базовые шаблоны: index.html (главная), about.html (о проекте), base.html (базовый шаблон), а также шаблоны для списка, детального просмотра, формы и подтверждения удаления.
— Подключена пагинация (по 10 записей).
— Все модели зарегистрированы в админке.

Практическая работа 2: Добавление REST API
Вариант 2: API для User и Organization

К существующему проекту добавлено REST API с использованием Django REST Framework.

Реализовано:
— Сериализаторы для User и Organization.
— Для User добавлено вычисляемое поле organizations — список названий организаций, где состоит пользователь.
— Представления на основе ModelViewSet — дают автоматические list, retrieve, create, update, delete.
— Дополнительный эндпоинт: GET /api/organization/many-employees/ — возвращает топ-10 организаций по сотрудникам.

Как запустить проект:

Создать и активировать виртуальное окружение:
python -m venv venv
venv\Scripts\activate
Установить зависимости:
pip install -r requirements.txt
Применить миграции:
python manage.py migrate
+ Создать суперпользователя:
python manage.py createsuperuser
Запустить сервер:
python manage.py runserver
